# Stateパターン
## パターンの理解
- 状態遷移はどちらに持たせてもいい
- Singletonにしてる
- 状態が複数あり、状態が変わると振る舞いが変わるものに使う
	- 組み合わせが多い
	- 状態が増える可能性がある
- コンテキスト側からは状態に関する処理はStateに完全に任せられる

## サンプル ファミレスの配膳ロボットシステム
ファミレスの猫の配膳ロボットのフローに適応しました。
- キッチンで待機　料理を乗せスイッチを押すと客席へ
- 移動中　スイッチは無視。客席へつくと配膳モードへ
- 提供中 料理がとられてスイッチが押されるとキッチンへ戻る
